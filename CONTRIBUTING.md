# Contributing to Web-Cam Eye Tracking for Code Comprehension

Thank you for your interest in contributing to this research tool.

## Before You Start
Please open a GitHub Issue describing what you want to contribute before 
writing any code. This avoids duplicate work and ensures your contribution 
fits the research scope.

## Types of Contributions

### Code stimuli
Add .py, .java, .cpp, .ts, or .js files to a /stimuli directory.
Each file should be a self-contained function or class (20–80 lines).
Include a comment at the top with: language, estimated complexity 
(low/medium/high), and whether it is human-written or AI-generated.

### AI-generated code samples
Pairs are most valuable: the same function written by a human developer 
and generated by an LLM (specify which model and prompt).
Name files: functionname_human.py and functionname_gpt4o.py for example.

### Session data contributions
Export your session data using python analyzer.py.
Anonymise participant identifiers before sharing.
Submit as a pull request adding files to a /community_sessions directory.

### Bug reports
Use the GitHub Issues tab. Include: OS, Python version, webcam model, 
and the exact error message.

## Pull Request Process
1. Fork the repository
2. Create a branch: git checkout -b feature/your-feature-name
3. Make your changes
4. Test with: python test_user_selection.py and python test_enhancements.py
5. Submit a pull request with a clear description of what changed and why

## Code Style
Follow PEP 8 for Python files.
Add docstrings to any new functions.
Do not commit calibration .pkl files or session JSON files.
